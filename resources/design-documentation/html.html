Estructura de los archivos front-end para git

CSS

Todos las páginas enlazan una hoja de estilos común llamada full_main_min_v2.css
Este css contiene los estilos de IDC/Games detallados en la hoja de estilos. Este full_main_min_v2.css además incluye:

  • Los estilos particulares de IDC/GAMES
  • El css de Bootstrap adaptado
  • El css de animate.sass.
  • El atlas de las banderas del menú en base 64 y sus posiciones
  • Las img de las texturas .bgDarkTexture y .bgLightTexture en base 64
  • Las instrucciones para el cambio de tipografía según lenguaje
El full_main_min_v2.css no se debe tocar.
Además del main.css, cada página tiene su css particular, cuyos estilos sobrescriben a main.css y que es llamada css.css
En este css.css escribiremos los estilos particulares de cada página. Este css.css es el que se sube al panel.

HTML

Cada página tiene las siguientes piezas de HTML que se suben como modelos separados al panel:

  • HTML --- Nav
  • HTML --- Main
  • HTML --- Footer
  • HTML --- Modal (Opcional)

Hay 2 tipos de menú (nav) distintos: el general (nav) y el de la página de juego (nav-game).

Para el menú de la página de juego, se cambia el color y la altura de este mediante las siguientes funciones:
blackMenu();
whiteMenu();

JS

Cada página tiene las siguientes piezas de Javascript que se suben por separado al panel:

  • JS --- js-ready.js ----> JS on ready
  • JS --- js-head.html ----> JS situado en el head del HTML
  • JS --- js-footer.html ----> JS situado en el footer del HTML

Hay una serie de scripts y librerías que ya están incluídas en un archivo global que comparten todos los modelos, como: jquery, popper.js, bootstrap.js, perfect-scrollbar y más.

Librerias, toolkits y plugins font-end utilizados en IDC/Games

Bootstrap

El diseño de IDC/Games está contruído sobre Bootstrap. Mediante SCSS se adaptaron las clases de Bootstrap para crear la hoja de estilos de IDC/Games. Consultando la documentación de Bootstrap y sus componentes se pueden sacar casi todos los elemento gráficos que componen IDC/Games. Utilizamos contenedores flex, las rejillas de Bootstrap y las utilities de spacing

Lottie

Para iconos animados como el del menú

Slick

Para los "carousels"

Animate.css

O animate.sass. Se utiliza para animaciones css cuando se carga la página. Las clases están incluidas en el main.css

Aos / Animated on scroll library

Animaciones en scroll. Utilizadas en la página de juego (game home), en las de categoría antiguas y alguna más. No está incluido en el main ni en los achivos globales, ya que es pesada y no se usa en todas las páginas. Hay que añadirla en los modelos en los que se incluya.

Perfect scrollbar

Para la barra lateral de scroll y otras.
